# Idea on using Multi-LoRAs obiting around an LLM to Catastrophic Forgetting
The general idea is to keep the central LLM unchanged, while surrounding it with a dozen LoRAs, each with an assigned weight and made to be orthogonal to each other. The system is made to be enclosed in an agent that does real-time observation, i.e., a robot, and perform continuous training. Every short moment, each LoRA will be queried with "summarize your observation and your last output," followed by real-time data received. At the same time, the LLM will be queried with "summarize your observation and the most recent user input," followed by real-time data received and user input. A machine then updates the relevant weight between the LoRA's and LLM's output and trains on the LoRA with the highest weight by an epoch.
